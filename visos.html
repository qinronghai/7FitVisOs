0. 7Fit 品牌与业务画像（策略锚点）

视觉系统必须服务业务模型：7Fit是评估驱动型功能训练工作室，而非传统办卡健身房。

机构本质（What）

  • 定位：体态评估驱动型训练机构（私教为主，小团课为辅）。
  • 方法：力量训练 + 普拉提 + 筋膜刀 + 运动康复协同。
  • 价值：功能恢复 + 塑形提升 + 慢性疼痛改善并行。
视觉含义：强调“专业与可信”，避免商业健身房常见的躁动与促销感。

核心业务（How）

模块7Fit重点
私教训练女性塑形、减脂、臀腿、马甲线、力量入门、功能训练
普拉提核心激活、呼吸控制、深层稳定肌、体态重建、产后恢复
筋膜刀松解肩颈/背部/下肢松解，运动恢复，慢性疼痛干预
体态矫正专项骨盆前倾、圆肩驼背、头前引、足弓塌陷、拇外翻等功能改善

1. 设计原则（必须统一）

清晰优先
任何页面先保证“3秒可扫描”，再追求视觉表现。
行动导向
每屏只保留1个主行动CTA，避免按钮竞争。
数据可信
指标使用等宽数字，对齐稳定，避免跳动与错读。
克制统一
颜色、圆角、间距不随页面自由发挥。
设计判断题通过标准
强调是否过多？品牌紫面积 ≤ 15%，且仅用于CTA、状态、关键数字。
层级是否明确？标题 / 标签 / 指标 / 内容 / 操作 五层清楚。
是否“一个品牌”感？组件圆角、边框、字体、阴影与主规范一致。
是否可交付开发？每个样式都能映射到token或组件变量。
典型问题来源：间距不足 + 强调过量 + 组件状态缺失。

2. Logo 规范

确保在数字端与印刷端保持同等识别强度。

安全区与最小尺寸

  • 安全区：四周 ≥ 0.5 ×“F”字高度。
  • 数字端最小宽度：120px；印刷端最小宽度：22mm。
  • 反白版本：深底优先白字，不加描边不加发光。
禁止：拉伸、旋转、渐变描边、重阴影、改字形、贴边摆放。

推荐占比（默认界面）
  • 中性色（黑/灰/白）：70%–85%
  • 品牌紫：8%–15%
  • 状态色（成功/警告/错误/信息）：≤8%
颜色用途边界
  • 紫色用于 CTA、重点标签、关键数据、选中态。
  • 正文、说明、长文档禁止大段紫色文本。
  • 错误状态需“颜色 + 图标/文案”双重表达。

5. 栅格、间距、圆角、阴影

空间系统（8pt 基线）

对象建议值说明
页面主区边距24–40留足呼吸感，避免边贴内容
模块间距32–56形成结构断点，利于扫描
卡片内边距16–24紧凑但不拥挤
表单项垂直距12–16阅读与输入节奏平衡
按钮组间距8–12操作不误触

形状与层级

  • Card：16/20 圆角（大容器用20）。
  • Input / Button：12 圆角（移动端统一手感）。
  • Tag / Badge：Pill 胶囊（999）。
  • 阴影：只用 3 级（Surface / Popup / Drawer），避免“每块都阴影”。
“高级感”不靠花样，靠统一与克制。

6. 动效规范

场景时长曲线规则
Hover / 按压反馈120msease-out快反馈，不拖沓
组件状态切换200msease-in-out平滑但不过度
抽屉 / 弹层入场280–320mscubic-bezier(.2,.8,.2,1)有重量感，避免眩晕

动效原则

  • 训练与数据页面：动效偏稳，重点在可读性。
  • 激励反馈（完成、达标）：动效偏快，强调正反馈。
  • 支持“减少动态”系统设置（尊重用户偏好）。

状态覆盖优先级（从高到低）

  • 任何状态不可破坏可读性（文字对比度优先）。

9. 响应式规范（P0）

明确断点、栅格、字体与间距缩放规则，保证移动端到大屏体验连贯。

断点宽度栅格间距策略排版/信息策略
Mobile< 7684列 / 边距16模块间距 24-32；卡片内边距 14-16单列优先；先KPI后细节；高频CTA固定底部可见。
Tablet768-11998列 / 边距20-24模块间距 32-40；卡片内边距 16-20双列信息；避免三栏拥挤；保留关键图表。
Desktop≥ 120012列 / 边距24-40模块间距 40-56；卡片内边距 20-24可用2-3列；信息并排对比；侧栏增强导航效率。

字体缩放基线

  • Mobile：H1 28 / H2 22 / Body 14
  • Tablet：H1 30 / H2 23 / Body 14-15
  • Desktop：H1 32 / H2 24 / Body 14-16
禁止仅缩小字体解决拥挤，优先重排信息结构。

响应式 Do / Don’t

  • Do：按优先级折叠低频信息（次级说明、扩展字段）。
  • Do：把对比数据改为纵向堆叠，避免横向挤压。
  • Don’t：移动端保留桌面三列表格原样。
  • Don’t：在小屏保留过长行文本（>32个汉字一行）。

10. 数据可视化规范（P0）

围绕训练评估与复评场景，统一图表选型、视觉编码与风险阈值表达。

场景推荐图表编码规则阈值表达避免误导
阶段趋势（体脂/围度）折线图主指标用品牌紫；比较组用中性灰目标线/警戒线用虚线标注并命名禁止截断Y轴夸大波动
多项能力对比分组条形图同维度同色；数值右对齐达标值使用浅色区间背景禁止混用正负色导致语义冲突
动作质量雷达雷达图（限5-6维）填充透明度 12%-18%最低合格圈层需标注文本维度过多会失真，超6维改条形图
风险分级分段色条 / 仪表绿-黄-红固定映射阈值写明来源（评估标准）禁止只给颜色不给说明文案

图表基础样式

  • 网格线：仅主网格，颜色用 Border Default。
  • 坐标轴文本：12px，色值 Text Secondary。
  • 数据标签：默认隐藏，hover或关键点显示。
  • 图例顺序：与业务优先级一致，不按字母排序。

风险语义映射（统一）

  • 安全（Green）：可推进训练。
  • 关注（Amber）：需降级或加强监控。
  • 风险（Red）：暂停高负荷并复评。
  • 所有颜色状态必须配套文本标签。
图表目标是帮助决策，不是制造视觉冲击。优先准确、可比、可复盘。

11. 页面模板规范（跨端一致）

核心页面模板（按业务闭环）

  • Assessment：评估页（体测、体态、功能测试、训练风险等级）。
  • Plan：定制计划页（阶段目标、每周频次、替代动作、升级条件）。
  • Session：课程执行页（动作组、RPE、疼痛反馈、即时调整）。
  • Review：复评页（前后对比、问题变化、下一阶段建议）。
  • Profile：会员档案（历史训练、依从性、注意事项）。

每页信息顺序（统一）

  1. 页面标题 + 标签（阶段 / 风险 / 目标）
  2. 核心KPI区（3–6个）
  3. 主体内容分组（最多3层）
  4. 风险与提示 Callout
  5. 主操作区（1主2次）

12. DOCX / PDF 导出规范（A4友好）

模块规范落地要点
页眉Logo + 文档类型 + 日期/会员编号统一页眉紫线，不做花哨背景。
首页摘要目标、阶段、关键风险、3–6个KPI一页看懂核心状态。
评估页体测数据 + 体态问题 + 功能测试结果必须出现“问题优先级排序”。
训练计划区按“松解→激活→主训→核心→拉伸”结构输出表格化，减少大段文本。
复评页阶段前后对比 + 下阶段迭代建议体现“执行—复评—迭代”闭环。
风险提示独立Callout区（左紫线）高风险动作替代建议必须出现。

13. 无障碍与合规边界

A11y硬性要求

  • 正文最小字号 14px（移动端）；不得低于12px。
  • 对比度：正文 ≥ 4.5:1，标题 ≥ 3:1。
  • 触控热区 ≥ 44×44px。
  • 状态不可仅靠颜色表达，必须有文本/图标辅助。

营销与健康表达边界

  • 建议用词：改善、支持、缓解、提升稳定性。
  • 避免用词：治疗、治愈、保证见效、一次解决。
  • 对疼痛/不适提示：出现锐痛、放射痛建议就医。

15. 治理机制与版本策略

版本语义（推荐）

  • Major：视觉原则或组件结构重大调整（破坏性变更）。
  • Minor：新增组件、增加变体、扩展模板。
  • Patch：修复token、对齐样式细节、不改结构。

发布流程（简化版）

  1. 设计提交变更说明（目标 / 影响 / 回滚方式）
  2. 与前端评审 token 映射
  3. 在模板页做视觉回归检查
  4. 更新 changelog 并标记生效版本

16. QA 发布清单

视觉一致性

  • 品牌紫占比 ≤ 15%
  • 圆角/边框/阴影符合规范
  • 主次按钮层级清晰
  • 字体层级不混乱

结构可读性

  • 标题—指标—内容—操作顺序正确
  • KPI数量 3–6，重点明确
  • 风险信息有独立入口
  • 页面无信息堆叠

交互与状态

  • 组件具备默认/悬停/按压/禁用
  • 输入错误反馈完整
  • 加载与空状态可识别
  • 动效时长符合标准

可访问性与合规

  • 最小触控热区 44×44
  • 对比度达标
  • 非颜色单一传达
  • 无“治愈/保证见效”等违规用词
